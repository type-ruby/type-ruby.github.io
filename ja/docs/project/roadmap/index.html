ロードマップ

T-Rubyは活発に開発中です。このロードマップでは、プロジェクトの現在の状態、今後の機能、長期的なビジョンを概説します。

プロジェクト状態​

実験的リリース

T-Rubyは現在実験的/アルファ状態です。コア機能は動作しますが、言語とツールは進化し続けています。バージョン間で破壊的変更が発生する可能性があります。本番環境での使用は自己責任でお願いします。

現在のバージョン： v0.1.0-alpha リリース： 2025年12月 ライセンス： MIT

コア機能ステータス​

✅ 完了（v0.1.0）​

型システム​

  • 基本型（String、Integer、Float、Bool、Symbol、nil）
  • ユニオン型（String | Integer）
  • オプショナル型（String?）
  • ArrayとHashジェネリクス（Array<T>、Hash<K, V>）
  • 変数と戻り値の型推論
  • 関数パラメータと戻り値型アノテーション
  • インスタンス変数型アノテーション
  • クラス変数型アノテーション
  • is_a?とnil?による型絞り込み
  • リテラル型（文字列、数値、シンボル、ブーリアンリテラル）

コンパイラ​

  • .trbから.rbへのコンパイル
  • 型消去（ランタイムオーバーヘッドなし）
  • .rbsファイル生成
  • デバッグ用ソースマップ
  • ウォッチモード（--watch）
  • 型チェックモード（--check）
  • ファイル位置付きエラーメッセージ

ジェネリクス​

  • ジェネリック関数（def func<T>(...)）
  • ジェネリッククラス（class Box<T>）
  • 複数の型パラメータ
  • 型パラメータ推論
  • ネストされたジェネリクス

高度な型​

  • 型エイリアス（type UserID = Integer）
  • ジェネリック型エイリアス（type Result<T> = T | nil）
  • インターセクション型（Printable & Comparable）
  • Proc型（Proc<T, R>）
  • void戻り値型
  • 戻らない関数のためのnever型
  • メソッドチェーンのためのself型

標準ライブラリ​

  • コアRuby型（File、Dir、Timeなど）
  • JSON型シグネチャ
  • 一般的なstdlibモジュール（CSV、Loggerなど）

  7. パターンマッチング型 - 型安全パターンマッチング
  8. ヌル安全性 - 厳格なヌルチェックモード

研究領域​

以下の高度な機能を積極的に研究しています：

1. エフェクト型​

型システムで副作用を追跡：

def read_file(path: String): String throws IOError
def calculate(x: Integer): Integer pure

2. 依存型​

値に依存する型：

def create_array<N: Integer>(size: N): Array<T>[N]
# 正確にN個の要素を持つ配列を返す

3. 線形型​

リソースが正確に一度だけ使用されることを保証：

def process_file(handle: File) consume: String
# この呼び出し後はhandleを使用できない

4. 行多相性​

柔軟なレコード型：

def add_id<T: { ... }>(obj: T): T & { id: Integer }

破壊的変更ポリシー​

現在のフェーズ（アルファ）​

  • どのリリースでも破壊的変更が発生する可能性
  • 可能な限り非推奨警告を提供
  • 主要な変更の移行ガイド
  • 変更ログにすべての破壊的変更を文書化

ベータフェーズ（将来）​

  • マイナーバージョンでのみ破壊的変更
  • 最低1ヶ月の非推奨期間
  • 包括的な移行ツール
  • 詳細なアップグレードガイド

安定フェーズ（v1.0+）​

  • パッチバージョンで破壊的変更なし
  • メジャーバージョンでのみ破壊的変更
  • 最低6ヶ月の非推奨期間
  • 自動化された移行ツール
  • 企業向けLTSリリース

ロードマップに影響を与える方法​

コミュニティの意見を歓迎します：

  1. 機能への投票 - GitHubでissueにスターを付けて関心を示す
  2. 機能リクエスト - 詳細な機能リクエストを開く
  3. コードを貢献 - ロードマップ項目のPRを提出
  4. フィードバックを共有 - 何がうまくいっていて何がうまくいっていないか教えてください
  5. ディスカッションに参加 - RFCプロセスに参加

プロジェクト指標​

現在の重点領域と成功指標：

品質指標​

  • 90%以上のテストカバレッジ
  • すべてのコア機能を文書化
  • 一般的なファイルで100ms未満の型チェック
  • 5%未満の偽陽性エラー率

採用指標​

  • 100+ GitHubスター
  • 10+コントリビューター
  • 5+本番プロジェクト
  • 月1000+ npmダウンロード

エコシステム指標​

  • 20+型定義付きgem
  • 3+エディタ拡張機能
  • 50+サンプルプロジェクト
  • アクティブなコミュニティフォーラム

長期ビジョン​

T-Rubyに対する我々のビジョン：

5年目標​

  • 型付きRuby開発のデフォルトの選択
  • トップgemの型がある成熟したエコシステム
  • 本番アプリケーションでの業界採用
  • 型とツールを貢献するアクティブなコミュニティ
  • プログラミング言語イノベーションの研究プラットフォーム

抱負​

  • 生産性を犠牲にせずにRuby開発をより安全に
  • 動的言語で段階的型付けが機能することを証明
  • 開発者を喜ばせるツールを構築
  • 歓迎的で包括的なコミュニティを育成
  • PL（プログラミング言語）コミュニティに研究成果を還元
