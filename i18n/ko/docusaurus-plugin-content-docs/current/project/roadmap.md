# 로드맵

T-Ruby는 활발히 개발 중입니다. 이 로드맵은 프로젝트의 현재 상태, 예정된 기능, 장기 비전을 설명합니다.

## 프로젝트 상태

:::caution 실험적 릴리스
T-Ruby는 현재 **실험적/알파** 상태입니다. 핵심 기능은 작동하지만, 언어와 도구는 계속 발전하고 있습니다. 버전 간에 호환되지 않는 변경이 발생할 수 있습니다. 프로덕션에서의 사용은 본인의 책임 하에 진행해 주세요.
:::

**현재 버전:** v0.1.0-alpha
**릴리스:** 2025년 12월
**라이선스:** MIT

## 기능 요청

커뮤니티 피드백을 적극적으로 수집하고 있습니다. 가장 많이 요청된 기능:

1. **더 나은 IDE 지원** - LSP 구현 (진행 중)
2. **Rails 타입** - 포괄적인 Rails 타입 정의 (계획됨)
3. **점진적 마이그레이션 도구** - Ruby를 T-Ruby로 자동 변환
4. **타입 정의 생성기** - 기존 Ruby에서 `.trb` 타입 생성
5. **성능 개선** - 대규모 프로젝트에서 더 빠른 컴파일
6. **더 나은 오류 메시지** - 더 도움이 되는 타입 오류 설명
7. **패턴 매칭 타입** - 타입 안전 패턴 매칭
8. **널 안전성** - 엄격한 널 검사 모드

## 연구 영역

다음 고급 기능을 적극적으로 연구하고 있습니다:

### 1. 이펙트 타입
타입 시스템에서 부작용 추적:
```ruby
def read_file(path: String): String throws IOError
def calculate(x: Integer): Integer pure
```

### 2. 의존 타입
값에 의존하는 타입:
```ruby
def create_array<N: Integer>(size: N): Array<T>[N]
# 정확히 N개의 요소를 가진 배열 반환
```

### 3. 선형 타입
리소스가 정확히 한 번만 사용되도록 보장:
```ruby
def process_file(handle: File) consume: String
# 이 호출 후에는 handle을 사용할 수 없음
```

### 4. 로우 다형성
유연한 레코드 타입:
```ruby
def add_id<T: { ... }>(obj: T): T & { id: Integer }
```

## 호환되지 않는 변경 정책

### 현재 단계 (알파)
- 모든 릴리스에서 호환되지 않는 변경이 발생할 수 있음
- 가능한 경우 폐기 예정 경고 제공
- 주요 변경에 대한 마이그레이션 가이드
- 변경 로그에 모든 호환되지 않는 변경 문서화

### 베타 단계 (향후)
- 마이너 버전에서만 호환되지 않는 변경
- 최소 1개월의 폐기 예정 기간
- 포괄적인 마이그레이션 도구
- 상세한 업그레이드 가이드

### 안정 단계 (v1.0+)
- 패치 버전에서 호환되지 않는 변경 없음
- 메이저 버전에서만 호환되지 않는 변경
- 최소 6개월의 폐기 예정 기간
- 자동화된 마이그레이션 도구
- 기업용 LTS 릴리스

## 장기 비전

T-Ruby에 대한 우리의 비전:

### 5년 목표
- 타입이 지정된 Ruby 개발의 **기본 선택**
- 상위 gem에 대한 타입이 있는 **성숙한 생태계**
- 프로덕션 애플리케이션에서의 **산업 채택**
- 타입과 도구를 기여하는 **활발한 커뮤니티**
- 프로그래밍 언어 혁신을 위한 **연구 플랫폼**

### 열망
- 생산성을 희생하지 않고 Ruby 개발을 더 안전하게 만들기
- 동적 언어에서 점진적 타이핑이 작동함을 증명
- 개발자를 기쁘게 하는 도구 구축
- 환영하고 포용적인 커뮤니티 육성
- PL 커뮤니티에 연구 결과 환원
