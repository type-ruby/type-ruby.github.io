# ロードマップ

T-Rubyの開発ロードマップはGitHubで管理されています。

## リンク

- [GitHubプロジェクトボード](https://github.com/orgs/type-ruby/projects/1)
- [マイルストーン](https://github.com/type-ruby/t-ruby/milestones)
- [ロードマップissue](https://github.com/type-ruby/t-ruby/labels/roadmap)

## ロードマップに影響を与える方法

コミュニティの意見を歓迎します：

1. **機能への投票** - GitHubでissueにスターを付けて関心を示す
2. **機能リクエスト** - [機能リクエストを開く](https://github.com/type-ruby/t-ruby/issues/new)

## 参加する

T-Rubyの未来を形作る手助けをしてください：

- **ドキュメント** - ドキュメントとサンプルを改善
- **テスト** - バグとエッジケースを報告
- **機能** - 機能を提案して実装
- **ツール** - エディタ拡張機能を構築
- **型** - stdlibとgemの型定義を追加
- **コミュニティ** - 質問に答え、チュートリアルを書く

始めるには[貢献ガイド](/docs/project/contributing)をご覧ください。

---

*ロードマップはGitHubで管理されており、コミュニティフィードバックと優先事項に基づいて変更される可能性があります。*
