로드맵

T-Ruby는 활발히 개발 중입니다. 이 로드맵은 프로젝트의 현재 상태, 예정된 기능, 장기 비전을 설명합니다.

프로젝트 상태​

실험적 릴리스

T-Ruby는 현재 실험적/알파 상태입니다. 핵심 기능은 작동하지만, 언어와 도구는 계속 발전하고 있습니다. 버전 간에 호환되지 않는 변경이 발생할 수 있습니다. 프로덕션에서의 사용은 본인의 책임 하에 진행해 주세요.

현재 버전: v0.1.0-alpha 릴리스: 2025년 12월 라이선스: MIT

핵심 기능 상태​

✅ 완료됨 (v0.1.0)​

타입 시스템​

  • 기본 타입 (String, Integer, Float, Bool, Symbol, nil)
  • 유니온 타입 (String | Integer)
  • 선택적 타입 (String?)
  • Array와 Hash 제네릭 (Array<T>, Hash<K, V>)
  • 변수와 반환 값에 대한 타입 추론
  • 함수 파라미터와 반환 타입 어노테이션
  • 인스턴스 변수 타입 어노테이션
  • 클래스 변수 타입 어노테이션
  • is_a?와 nil?을 사용한 타입 좁히기
  • 리터럴 타입 (문자열, 숫자, 심볼, 불리언 리터럴)

컴파일러​

  • .trb에서 .rb 컴파일
  • 타입 지우기 (런타임 오버헤드 없음)
  • .rbs 파일 생성
  • 디버깅을 위한 소스 맵
  • 감시 모드 (--watch)
  • 타입 검사 모드 (--check)
  • 파일 위치가 포함된 오류 메시지

제네릭​

  • 제네릭 함수 (def func<T>(...))
  • 제네릭 클래스 (class Box<T>)
  • 여러 타입 파라미터
  • 타입 파라미터 추론
  • 중첩된 제네릭

고급 타입​

  • 타입 별칭 (type UserID = Integer)
  • 제네릭 타입 별칭 (type Result<T> = T | nil)
  • 교차 타입 (Printable & Comparable)
  • Proc 타입 (Proc<T, R>)
  • void 반환 타입
  • 반환하지 않는 함수를 위한 never 타입
  • 메서드 체이닝을 위한 self 타입

표준 라이브러리​

  • 코어 Ruby 타입 (File, Dir, Time 등)
  • JSON 타입 시그니처
  • 일반적인 stdlib 모듈 (CSV, Logger 등)

기능 요청​

커뮤니티 피드백을 적극적으로 수집하고 있습니다. 가장 많이 요청된 기능:

  1. 더 나은 IDE 지원 - LSP 구현 (진행 중)
  2. Rails 타입 - 포괄적인 Rails 타입 정의 (계획됨)
  3. 점진적 마이그레이션 도구 - Ruby를 T-Ruby로 자동 변환
  4. 타입 정의 생성기 - 기존 Ruby에서 .trb 타입 생성
  5. 성능 개선 - 대규모 프로젝트에서 더 빠른 컴파일
  6. 더 나은 오류 메시지 - 더 도움이 되는 타입 오류 설명
  7. 패턴 매칭 타입 - 타입 안전 패턴 매칭
  8. 널 안전성 - 엄격한 널 검사 모드

연구 영역​

다음 고급 기능을 적극적으로 연구하고 있습니다:

1. 이펙트 타입​

타입 시스템에서 부작용 추적:

def read_file(path: String): String throws IOError
def calculate(x: Integer): Integer pure

2. 의존 타입​

값에 의존하는 타입:

def create_array<N: Integer>(size: N): Array<T>[N]
# 정확히 N개의 요소를 가진 배열 반환

3. 선형 타입​

리소스가 정확히 한 번만 사용되도록 보장:

def process_file(handle: File) consume: String
# 이 호출 후에는 handle을 사용할 수 없음

4. 로우 다형성​

유연한 레코드 타입:

def add_id<T: { ... }>(obj: T): T & { id: Integer }

호환되지 않는 변경 정책​

현재 단계 (알파)​

  • 모든 릴리스에서 호환되지 않는 변경이 발생할 수 있음
  • 가능한 경우 폐기 예정 경고 제공
  • 주요 변경에 대한 마이그레이션 가이드
  • 변경 로그에 모든 호환되지 않는 변경 문서화

베타 단계 (향후)​

  • 마이너 버전에서만 호환되지 않는 변경
  • 최소 1개월의 폐기 예정 기간
  • 포괄적인 마이그레이션 도구
  • 상세한 업그레이드 가이드

안정 단계 (v1.0+)​

  • 패치 버전에서 호환되지 않는 변경 없음
  • 메이저 버전에서만 호환되지 않는 변경
  • 최소 6개월의 폐기 예정 기간
  • 자동화된 마이그레이션 도구
  • 기업용 LTS 릴리스

로드맵에 영향을 미치는 방법​

커뮤니티 의견을 환영합니다:

  1. 기능 투표 - GitHub에서 이슈에 별표를 눌러 관심 표시
  2. 기능 요청 - 상세한 기능 요청 열기
  3. 코드 기여 - 로드맵 항목에 대한 PR 제출
  4. 피드백 공유 - 무엇이 효과적이고 무엇이 아닌지 알려주기
  5. 토론 참여 - RFC 프로세스에 참여

릴리스 일정​

예측 가능한 릴리스를 목표로 합니다:

  • 마이너 릴리스 (v0.x.0): 분기별, 새 기능
  • 메이저 릴리스 (vX.0.0): 연간, 호환되지 않는 변경

장기 비전​

T-Ruby에 대한 우리의 비전:

5년 목표​

  • 타입이 지정된 Ruby 개발의 기본 선택
  • 상위 gem에 대한 타입이 있는 성숙한 생태계
  • 프로덕션 애플리케이션에서의 산업 채택
  • 타입과 도구를 기여하는 활발한 커뮤니티
  • 프로그래밍 언어 혁신을 위한 연구 플랫폼

열망​

  • 생산성을 희생하지 않고 Ruby 개발을 더 안전하게 만들기
  • 동적 언어에서 점진적 타이핑이 작동함을 증명
  • 개발자를 기쁘게 하는 도구 구축
  • 환영하고 포용적인 커뮤니티 육성
  • PL 커뮤니티에 연구 결과 환원
